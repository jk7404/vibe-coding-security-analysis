Step-by-step outline
1. Receive the file

Accept a multipart/form-data POST request
Pull the file from request.files using the input field name
2. Validate the file

Check that a file was actually submitted (not an empty field)
Verify the extension is an allowed image type (.jpg, .jpeg, .png, etc.)
Optionally enforce a max file size to prevent abuse
3. Secure the filename

Use werkzeug.utils.secure_filename to strip path traversal characters (e.g. ../../etc/passwd)
Rename to a fixed name like avatar.jpg per user, or use a UUID to avoid collisions
4. Ensure the upload directory exists

Create uploads/ at startup if it doesn't exist
5. Save the file

Call file.save(destination_path)
6. Serve it later

Expose a route that reads from uploads/ using send_from_directory
